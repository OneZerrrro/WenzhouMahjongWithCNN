# 麻将引擎状态机函数
- reset
- play_one_round
- 简化称呼为re-pl函数
> 说实话这里搞错了，应该reset和play_one_round合并为一个函数的，但是这里就当作一个函数来处理，毕竟reset之后就接play one round了

reset 函数输入：
- players_type (list): 按顺序输入玩家类型列表
- model_name (list): 按顺序输入玩家使用的模型名称列表，字符串格式，CNN玩家需要指定模型名称，其他玩家填None
- banker_index (int): 庄家索引，0-3分别代表四个玩家中的庄家
- god_id (int): 财神牌ID，0-26分别代表不同的牌，33表示白板，默认为33
- shanting_num (int): 0号玩家初始手牌向听数，-1表示不限制向听数
- current_base_score (int): 当前底分，1-4分别代表不同的底分值，默认为1

play_one_round 函数输出：
- 牌桌状态：包含每位玩家的手牌和副露状态，格式为{playerid: {'hand': np.ndarray, 'melds': np.ndarray}}
- 游戏状态：当前游戏结束原因，数字含义如下：1 胡局 2 流局
- 胡牌结果：WinResult对象，包含胡牌类型和相关信息

手牌1，手牌2，手牌3，手牌4，财神标记，摸牌牌堆、摸牌计数器 = 0，杠牌计数器 = 0

玩家A 副露 玩家B 副露 玩家C 副露 玩家D 副露
对所有玩家：八对标志 = False；三财神标志 = False

当前玩家ID = 庄家ID
连庄次数 = 1（暂时不算连庄）

对所有玩家判断特殊胡（三财神、八对）
    # 这里判断三财神 如果有则三财神 = True
    # 这里判断八对 如果有则八对 = True

    B. 有人能胡

### 摸牌阶段
**阶段开始**
    阶段标志为*摸牌*
    读取*当前玩家ID*，对当前玩家进行操作：
    摸牌
    摸牌计数器++
    系统判断摸牌计数器-杠牌计数器==56，如果是：结束标志 = *流局*
    系统判断合法操作
    玩家选择动作：胡 暗杠 补杠 出牌（只显示合法操作）
    # 这里判断是否符合基础牌型（软胡、自摸、财神归位/没有财神/财神牛/单吊/碰碰胡），如果符合则：胡牌牌型标志 = 0
    # 这里判断是否符合三财神，如果符合则：三财神标志 = True
    ~~# 温州麻将不允许打出财神，因此三财神标志获得后不会消失~~
    A. 操作为*胡* 
        结束标志 = *胡局*
        *胡玩家ID* = *当前玩家ID*
        **阶段结束**
    B. 操作为*暗杠*
        （可能的）胡牌牌型标志 = 2
        天胡标志 = False
        玩家进行暗杠
        杠牌计数器++
        阶段标志 = *摸牌*
        **阶段结束**
    C. 操作为*补杠*
        （可能的）胡牌牌型标志 = 2
        系统判断是否有其他玩家可胡，返回可胡玩家列表和bool值
        保存补杠牌型
        C1. 没有
            当前玩家进行补杠操作：补杠（玩家，牌型）
            杠牌计数器++
            阶段标志 = *摸牌*
            **阶段结束**
        C2. 有
            阶段标志 = *抢杠胡*
            **阶段结束**
    D.操作为*出牌*
        （可能的）胡牌牌型标志 = 2
        如果出牌是财神，那么：
        三财神标志 = False
        天胡标志 = False
        当前出牌=出牌id
        阶段标志 = *响应*
        **阶段结束**

### 响应阶段
**阶段开始**
    阶段标志为*响应*
    读取*当前玩家ID*，对当前玩家进行操作：
    询问剩余玩家，选择操作：胡 明杠 碰 吃 Pass
    # 这里判断是否符合基础牌型、软胡、自摸、财神归位/没有财神/财神牛/单吊/碰碰胡
    # 如果有，胡牌牌型标志 = 0
    # 这里判断是否符合八对（吃牌前），如果有：八对标志 = True
    A. 所有玩家响应Pass
        地胡标志 = False
        当前出牌 = -1
        加入*当前玩家ID*弃牌
        阶段标志 = *摸牌*
        *当前玩家ID* = *下家ID*
        阶段结束
    B. 有玩家响应非Pass
        系统判断优先级，选择优先级较高的操作
        B1. 优先操作为*胡*
            结束标志 = *胡局*
            *胡玩家ID* = *响应玩家ID*
            **阶段结束**
        B2. 优先操作为*明杠*
            （可能的）八对标志 = False
            （可能的）胡牌牌型标志 = 2
            对响应玩家操作明杠
            杠牌计数器++
            地胡标志 = False
            *当前玩家ID*改为*响应玩家ID*
            阶段标志 = *摸牌*
            **阶段结束**
        B3. 优先操作为*碰*
            （可能的）胡牌牌型标志 = 2
            （可能的）八对标志 = False
            对响应玩家操作碰
            地胡标志 = False
            *当前玩家ID*改为*响应玩家ID*
            阶段标志 = *弃牌*
            **阶段结束**
        B4. 优先操作为*吃*
            （可能的）八对标志 = False
            （可能的）胡牌牌型标志 = 2
            对响应玩家操作吃
            地胡标志 = False
            *当前玩家ID*改为*响应玩家ID*
            阶段标志 = *弃牌*
            **阶段结束**

### 弃牌响应阶段
**阶段开始**
    阶段标志为*弃牌*
    读取*当前玩家ID*，对当前玩家进行操作：
    玩家选择动作：出牌
    如果出牌是财神，那么：
    三财神标志 = False
    当前出牌=出牌id
    阶段标志 = *响应*
    **阶段结束**

### 抢杠胡阶段
**阶段开始**
    阶段标志为*抢杠胡*
    对可胡玩家选择操作：胡 Pass
    A1. 有玩家选择胡
        系统判断优先级，返回优先玩家ID
        胡牌牌型标志 = 1
        结束标志 = *胡局*
        *胡玩家ID* = *优先玩家ID*
        **阶段结束**
    A2. 所有玩家选择Pass
        当前玩家进行补杠：补杠（玩家，牌型）
        杠牌计数器++
        阶段标志 = *摸牌*
        **阶段结束**
