# 单存档状态机代码

状态机：
|0|1|2|3|4|
|---|---|---|---|---|
|单存档主界面|开始今天游戏|查看当前存档信息|保存并回到上一级菜单|保存并退出|

状态机函数输入：存档名称
状态机函数输出：是否直接退出
状态配置：
- 状态机标志
- Game状态
- 存档名称
- 存档信息
- 游戏选项，包括：
    - 初始游戏选项
        - players_type (list)
        - model_name (list)
        - banker_index (int)
    - 可变游戏选项
        - god_id (int)
        - shanting_num (int)
        - current_base_score (int)
- 分数计算相关变量
    - 牌桌状态：包含每位玩家的手牌和副露状态
    - 游戏状态：当前游戏结束原因
    - 胡牌结果：WinResult对象

## 重置单存档状态机
输入：存档名称
自动加载保存在存档内的信息

## 单存档状态机主函数
### 单存档主界面
**阶段开始**
绘制主界面
接收选项（1，2，3，4）
状态标志 = ***输入***
**阶段结束**
### 开始今天游戏（嵌套状态机）
**阶段开始**
绘制开始今天游戏界面
接收选项：
- 以下选项需要手动输入
    - 参与游戏的玩家名称

自动配置以下初始选项：
- players_type (list): 根据游戏玩家名称按顺序配置玩家类型列表
- model_name (list): 根据游戏玩家名称配置使用的模型名称列表，字符串格式，CNN玩家需要指定模型名称，其他玩家填None
- banker_index (int): 随机选择庄家索引，0-3分别代表四个玩家中的庄家

配置：
- Game状态 == 0

进入**状态机Game状态**：
- **Game状态 == 0**
    接收选项：（1，2，3）
    Game状态 == 接收状态
- **Game状态 == 1**
    自动配置以下可变选项：
    - god_id (int): 财神牌ID，0-26分别代表不同的牌，33表示白板，默认为33
    - shanting_num (int): 0号玩家初始手牌向听数，-1表示不限制向听数（默认为-1）
    - current_base_score (int): 当前底分，1-4分别代表不同的底分值，默认为1

    以当前配置项调用**re-pl**函数
    返回以下变量：
    - 牌桌状态：包含每位玩家的手牌和副露状态
    - 游戏状态：当前游戏结束原因
    - 胡牌结果：WinResult对象

    自动更新分数信息并保存
    接收选项：（1，2，3）
    Game状态 == 接收状态

- **Game状态 == 2**
    绘制当前四人分数信息
    接收**任意键**
    Game状态 == 0

- **Game状态 == 3**
    状态标志 = ***0***
    退出**状态机Game状态**

**阶段结束**
